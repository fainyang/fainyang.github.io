ELK 日志收集系统快速搭建

发表于 2019-03-25 | 更新于 2019-03-26 | 分类于 Linux | 评论数： | 阅读次数：

ELK 是一个分布式、可扩展、实时的搜索与数据分析引擎。目前我在工作中只用来收集 server 的 log, 开发锅锅们 debug 的好助手。

单节点 ELK

如果你想快速的搭建单节点 ELK, 那么使用 docker 方式肯定是你的最佳选择。使用三合一的镜像，文档详情: https://elk-docker.readthedocs.io/
注意：安装完 docker, 记得设置 mmap counts 至少 262144 命令为 sysctl -w vm.max_map_count=262144
单节点的机器，不必暴露 9200(Elasticsearch JSON interface) 和 9300(Elasticsearch transport interface) 端口。

index 的规则 https://www.elastic.co/guide/en/elasticsearch/reference/6.4/indices-create-index.html

  • Lowercase only
  • Cannot include \, /, *, ?, “, <, >, |, ` ` (space character), ,, #
  • Indices prior to 7.0 could contain a colon (:), but that’s been deprecated and won’t be supported in 7.0+
  • Cannot start with -, _, +
  • Cannot be . or ..
  • Cannot be longer than 255 bytes (note it is bytes, so multi-byte characters will count towards the 255 limit faster)
